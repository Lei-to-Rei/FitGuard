  The smartwatch collects 60-second sequences of PPG (heart rate) and

  2. Feature Extraction (SequenceProcessor)

  - SpO2 (16-18): mean, min, std

  - Steps (28-29): total steps, cadence

в—Ҹ Here's what you already have in place and what you can do:

  ---
  What You Already Have

  - Every session writes features.jsonl вҖ” all 30 features per window with
  timestamps
  - UserProfile stores restingHeartRateBpm and restingHrvRmssd
  - The scaler is just mean[] + std[] arrays used for z-score normalization

  The Simplest On-Device Personalization: Build a User-Specific Scaler

  The base model was trained on population-level data. The scaler normalizes
  inputs using population mean/std. But a user with resting HR of 55 vs one with
   85 will produce very different normalized values вҖ” the model sees them
  differently even at the same relative effort level.

  The fix: after enough sessions, compute the user's own mean/std across all
  their features.jsonl data, save it as their personalized scaler. No retraining
   needed вҖ” same model, user-adapted normalization.

  How it would work:

  1. Accumulate вҖ” you're already saving features.jsonl per session
  2. Trigger вҖ” after N sessions (e.g., 5), compute a user scaler
  3. Compute вҖ” read all the user's features.jsonl files, calculate mean and std
  for each of the 30 features
  4. Save вҖ” write to personalized/user_{userId}_scaler.json (same format as the
  global one, with same thresholds/level names)
  5. Load вҖ” initializePersonalized picks it up next session, using the base
  model + user scaler

  This way:
  - No model retraining needed
  - The base fatigue_model.tflite is still used
  - Only the normalization adapts to the individual
  - HR/HRV baselines from UserProfile could seed initial values before enough
  session data exists

  Want me to implement this? It would involve adding a
  generatePersonalizedScaler() method in FatigueDetector (or a utility) that
  reads the user's accumulated JSONL files and writes the scaler, then calling
  it after sessions reach the threshold.
